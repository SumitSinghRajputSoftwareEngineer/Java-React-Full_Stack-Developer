# Logging Levels

> **Goal:** Understand Logging Levels in depth, why they exist, the purpose of each level (TRACE, DEBUG, INFO, WARN, ERROR, OFF), when to use them, logging hierarchy, real-world examples, production best practices, and interview concepts.

---

# 1. What Are Logging Levels?

Not every log message has the same importance.

Consider the following events:

```text
User Logged In
Payment Successful
Database Connection Failed
Variable Value Changed
```

Some are:

```text
Informational
```

while others are:

```text
Critical Errors
```

To categorize logs based on severity, logging frameworks use:

```text
Logging Levels
```

---

## Definition

> Logging Levels define the severity and importance of log messages generated by an application.

---

## Why Logging Levels Exist?

Without logging levels:

```text
All Logs Look Identical
```

Developers would struggle to identify:

```text
Critical Issues
Warnings
Debug Information
```

---

## Example

Bad:

```text
User Logged In
Database Failed
Variable x = 10
Payment Successful
```

Everything looks the same.

---

With Logging Levels:

```text
INFO  User Logged In

DEBUG Variable x = 10

ERROR Database Failed

INFO  Payment Successful
```

Much easier to understand.

---

# 3. TRACE Level

TRACE is the most detailed logging level.

---

## Purpose

Used to track:

```text
Method Entry
Method Exit
Execution Flow
Variable Changes
Detailed Diagnostics
```

---

## Example

```java
log.trace("Entering calculateTax()");
```

---

Output:

```text
TRACE Entering calculateTax()
```

---

## Real Example

```java
public double calculateTax(double salary) {

    log.trace("Method Started");

    double tax = salary * 0.2;

    log.trace("Tax Calculated");

    return tax;
}
```

---

## When to Use TRACE?

Use for:

```text
Detailed Debugging
Framework Development
Complex Algorithm Analysis
Execution Flow Tracking
```

---

## When NOT to Use TRACE?

Avoid in production because:

```text
Huge Number of Logs
Storage Issues
Performance Impact
```

---

# TRACE Use Cases

```text
Method Entry/Exit
Loop Execution
Thread Execution Tracking
Framework Internals
```

---

# 4. DEBUG Level

DEBUG is intended for developers.

---

## Purpose

Used for:

```text
Development
Testing
Troubleshooting
```

---

## Example

```java
log.debug("Customer ID = {}", customerId);
```

---

Output:

```text
DEBUG Customer ID = 101
```

---

## Real Example

```java
log.debug("Order Request Received");

log.debug("Customer Id: {}", customerId);

log.debug("Products Count: {}", count);
```

---

## When to Use DEBUG?

Use for:

```text
Variable Values
Business Logic Validation
API Requests
Intermediate Processing Steps
```

---

## When NOT to Use DEBUG?

Avoid enabling DEBUG globally in production.

Reason:

```text
Large Log Volume
Disk Consumption
Performance Impact
```

---

# DEBUG Use Cases

```text
API Request Validation
Business Logic Tracking
Data Transformation
Input/Output Inspection
```

---

# 5. INFO Level

INFO is the most commonly used logging level.

---

## Purpose

Records normal business events.

---

## Example

```java
log.info("Order Created Successfully");
```

---

Output:

```text
INFO Order Created Successfully
```

---

# What Should Be Logged at INFO?

```text
Application Startup
Application Shutdown
Order Creation
Payment Success
User Login
User Logout
Business Events
```

---

## Real Example

```java
log.info("User Registered Successfully");

log.info("Payment Completed");

log.info("Invoice Generated");
```

---

# INFO Use Cases

```text
Business Events
Application Lifecycle
Successful Operations
Major Process Completion
```

---

# Why INFO Is Important?

Production monitoring relies heavily on:

```text
INFO Logs
```

because they provide:

```text
Business Visibility
Application Visibility
Operational Insights
```

---

# 6. WARN Level

WARN indicates something unexpected happened.

However:

```text
Application Can Continue
```

---

## Purpose

Warn developers or operators about potential problems.

---

## Example

```java
log.warn("Payment Gateway Response Delayed");
```

---

Output

```text
WARN Payment Gateway Response Delayed
```

---

## Real Examples

```java
log.warn("Disk Usage Above 80%");

log.warn("Retry Attempt 3");

log.warn("API Response Time Exceeded Threshold");
```

---

## When to Use WARN?

Use when:

```text
Unexpected Situation
Performance Issue
Recoverable Problem
Fallback Activated
```

---

# WARN Use Cases

```text
Slow Database Queries
Low Disk Space
Retry Operations
Deprecated API Usage
Missing Optional Data
```

---

# WARN vs ERROR

WARN:

```text
Something Wrong
Application Continues
```

ERROR:

```text
Operation Failed
```

---

# 7. ERROR Level

ERROR indicates a serious failure.

---

## Purpose

Used when an operation cannot complete successfully.

---

## Example

```java
log.error("Database Connection Failed");
```

---

Output:

```text
ERROR Database Connection Failed
```

---

## Real Examples

```java
log.error("Payment Processing Failed");

log.error("Database Connection Failed");

log.error("File Upload Failed");
```

---

## Exception Logging

Best Practice:

```java
log.error(
    "Error while creating order",
    exception
);
```

---

This records:

```text
Message
Stack Trace
Root Cause
```

---

# ERROR Use Cases

```text
Database Failure
Payment Failure
Service Unavailable
Unhandled Exceptions
Critical Application Errors
```

---

# Characteristics of ERROR

```text
Requires Investigation
May Impact Users
May Trigger Alerts
```

---

# 8. OFF Level

OFF disables logging completely.

---

## Purpose

Turns off all logging.

---

Example:

```properties
logging.level.root=OFF
```

---

Result:

```text
No Logs Generated
```

---

# When to Use OFF?

Rarely used.

Possible scenarios:

```text
Performance Testing
Temporary Troubleshooting
Special Environments
```

---

# 9. How Logging Hierarchy Works

This is one of the most important interview topics.

---

Suppose:

```properties
logging.level.root=INFO
```

---

Framework automatically allows:

```text
INFO
WARN
ERROR
```

---

and blocks:

```text
DEBUG
TRACE
```

---

# Example

Application:

```java
log.trace("Trace");

log.debug("Debug");

log.info("Info");

log.warn("Warn");

log.error("Error");
```

---

Configuration:

```properties
logging.level.root=INFO
```

---

Output:

```text
INFO Info

WARN Warn

ERROR Error
```

---

TRACE and DEBUG are ignored.

---

# Hierarchy Table

| Configured Level | Visible Logs |
|-----------------|-------------|
| TRACE | TRACE, DEBUG, INFO, WARN, ERROR |
| DEBUG | DEBUG, INFO, WARN, ERROR |
| INFO | INFO, WARN, ERROR |
| WARN | WARN, ERROR |
| ERROR | ERROR |
| OFF | None |

# 10. Choosing Appropriate Log Levels

Choosing the correct level is extremely important.

---

## Use TRACE For

```text
Method Entry
Method Exit
Framework Internals
Detailed Diagnostics
```

---

## Use DEBUG For

```text
Variable Values
Intermediate Processing
Developer Troubleshooting
```

---

## Use INFO For

```text
Successful Operations
Business Events
Application Startup
Application Shutdown
```

---

## Use WARN For

```text
Recoverable Issues
Slow Responses
Retries
Threshold Breaches
```

---

## Use ERROR For

```text
Failed Operations
Exceptions
Database Failures
External Service Failures
```

---

# Decision Tree

```text
Need Detailed Flow?
      ↓
    TRACE

Need Developer Information?
      ↓
    DEBUG

Normal Business Event?
      ↓
     INFO

Unexpected But Recoverable?
      ↓
     WARN

Operation Failed?
      ↓
    ERROR
```

# 11. Production Logging Strategy

A common industry practice.

---

## Development Environment

```properties
logging.level.root=DEBUG
```

Reason:

```text
Developers Need More Details
```

---

## Testing Environment

```properties
logging.level.root=INFO
```

Reason:

```text
Focus On Business Flow
```

---

## Production Environment

```properties
logging.level.root=INFO
```

or

```properties
logging.level.root=WARN
```

Reason:

```text
Avoid Excessive Logs
Reduce Storage Usage
Improve Performance
```

---

# Recommended Levels

| Environment | Log Level |
|-------------|------------|
| Local Development | DEBUG |
| QA | INFO |
| UAT | INFO |
| Production | INFO / WARN |
| Performance Testing | WARN |
| Troubleshooting Production | DEBUG (Temporary) |

# 12. Common Logging Level Mistakes

---

## Mistake 1

Logging Business Events As ERROR

Bad:

```java
log.error("User Logged In");
```

Correct:

```java
log.info("User Logged In");
```

---

## Mistake 2

Using INFO For Debugging

Bad:

```java
log.info("Variable x = {}", x);
```

Correct:

```java
log.debug("Variable x = {}", x);
```

---

## Mistake 3

Using ERROR For Warnings

Bad:

```java
log.error("Retrying Request");
```

Correct:

```java
log.warn("Retrying Request");
```

---

## Mistake 4

Enabling TRACE In Production

Can cause:

```text
Massive Log Files
Performance Problems
Storage Issues
```
