# Correlation ID Logging

> **Goal:** Understand Correlation IDs, why they are critical in Microservices architectures, how they enable request tracing across multiple services, implementation using Spring Boot Filters and Interceptors, production best practices, and interview concepts.

---

# 1. What is a Correlation ID?

One of the most important concepts in modern distributed systems.

---

## Definition

> A Correlation ID is a unique identifier attached to a request that travels across multiple services, allowing all logs related to that request to be tracked and correlated.

---

# Simple Definition

Think of a Correlation ID as:

```text
Tracking Number
```

for a request.

---

Example:

```text
CORR-12345
```

---

Every log generated by that request contains:

```text
CORR-12345
```

---

# Example

Request:

```text
Create Order
```

Correlation ID:

```text
CORR-12345
```

---

Logs:

```text
[CORR-12345]
Order Received

[CORR-12345]
Payment Started

[CORR-12345]
Inventory Updated

[CORR-12345]
Order Completed
```

---

Now the complete request journey is visible.

---

# Why Is It Called Correlation ID?

Because it helps:

```text
Correlate

Link

Associate
```

all logs generated by the same request.

---

```text
   ↓
```


```text
Unique Correlation ID
```

---


---

Now every log contains:

```text

```

# 3. Correlation ID vs Request ID

Very common interview question.

---

# Request ID

Used within:

```text
Single Application
```

---

Example:

```text
REQ-1001
```

---

Tracks:

```text
Request Inside One Service
```

---

# Correlation ID

Used across:

```text
Multiple Services
```

---

Example:

```text
CORR-1001
```

---

Tracks:

```text
Entire Distributed Request
```

---

# Comparison

| Request ID | Correlation ID |
|------------|---------------|
| Single Service | Multiple Services |
| Local Tracking | End-To-End Tracking |
| Simpler | More Powerful |
| Service Scope | System Scope |

---

# Example

Request ID:

```text
Tracks One Service
```

---

Correlation ID:

```text
Tracks Entire Request Journey
```

# 7. Implementation Using Filter

Most common Spring Boot implementation.

---

## Why Use Filter?

Filters execute:

```text
Before Controller

After Controller
```

---

Perfect place to:

```text
Generate Correlation ID
```

---

# Request Flow

```text
Request

   ↓

Filter

   ↓

Controller

   ↓

Response
```

---

# Example

```java
@Component
public class CorrelationIdFilter
        implements Filter {

    @Override
    public void doFilter(
            ServletRequest request,
            ServletResponse response,
            FilterChain chain) {

        String correlationId =
                UUID.randomUUID()
                    .toString();

        MDC.put(
            "correlationId",
            correlationId
        );

        try {

            chain.doFilter(
                request,
                response
            );

        } catch (Exception ex) {

            throw new RuntimeException(ex);

        } finally {

            MDC.clear();

        }
    }
}
```

---

# Flow

```text
Request Arrives

      ↓

Generate Correlation ID

      ↓

Store In MDC

      ↓

Controller Executes

      ↓

Logs Generated

      ↓

MDC Cleared
```

---

# Why Use finally?

Ensures:

```text
MDC Cleanup
```

always occurs.

---

# 8. Implementation Using Interceptor

Another common approach.

---

## Why Use Interceptor?

Provides:

```text
Request Lifecycle Hooks
```

---

Useful for:

```text
Logging

Authentication

Correlation IDs
```

---

# Flow

```text
Request

   ↓

Interceptor

   ↓

Controller

   ↓

Response
```

---

# Example

```java
@Component
public class CorrelationInterceptor
        implements HandlerInterceptor {

    @Override
    public boolean preHandle(
            HttpServletRequest request,
            HttpServletResponse response,
            Object handler) {

        String correlationId =
                UUID.randomUUID()
                    .toString();

        MDC.put(
            "correlationId",
            correlationId
        );

        return true;
    }

    @Override
    public void afterCompletion(
            HttpServletRequest request,
            HttpServletResponse response,
            Object handler,
            Exception ex) {

        MDC.clear();
    }
}
```

---

# Flow

```text
preHandle()

     ↓

Generate Correlation ID

     ↓

Store In MDC

     ↓

Controller

     ↓

Logs

     ↓

afterCompletion()

     ↓

Clear MDC
```

---

# Filter vs Interceptor

| Filter | Interceptor |
|----------|-------------|
| Servlet Level | Spring MVC Level |
| Runs Earlier | Runs After Filter |
| Works For All Requests | Works For MVC Requests |
| Framework Independent | Spring Specific |

---

# Which One Is Preferred?

Most organizations use:

```text
Filter
```

for correlation IDs because it executes:

```text
Earlier In Request Lifecycle
```

# 9. Correlation ID Propagation

Critical in Microservices.

---

# Problem

Service A generates:

```text
CORR-12345
```

---

Calls:

```text
Service B
```

---

If Service B generates:

```text
CORR-99999
```

tracking breaks.

---

# Solution

Forward existing Correlation ID.

---

Example Header:

```http
X-Correlation-ID:
CORR-12345
```

---

Service B receives:

```text
CORR-12345
```

---

Stores in MDC.

---

Logs:

```text
[CORR-12345]
```

---

All services now share:

```text
Same Correlation ID
```

---

# Distributed Tracing Flow

```text
Client

   ↓

API Gateway

   ↓

Order Service

   ↓

Payment Service

   ↓

Inventory Service

All Use

CORR-12345
```

# 10. Best Practices

---

## Use UUID

Recommended:

```java
UUID.randomUUID()
```

---

Reason:

```text
Globally Unique
```

---

## Store In MDC

Recommended:

```java
MDC.put(
    "correlationId",
    value
);
```

---

## Include In Log Pattern

Example:

```xml
%X{correlationId}
```

---

## Propagate Across Services

Pass through:

```http
X-Correlation-ID
```

header.

---

## Clear MDC

Always:

```java
MDC.clear();
```

---

---

Problem:

```text
Correlation ID Leakage
```

## Generating New IDs Per Service

Problem:

```text
Broken Traceability
```

---

## Forgetting Log Pattern

Problem:

```text
Stored But Not Displayed
```

---

## Not Propagating Header

Problem:

```text
Distributed Tracking Fails
```

# 12. Interview Questions

### Q1. What is a Correlation ID?

A unique identifier used to track a request across multiple services.

---

### Q2. Why do we need Correlation IDs?

For:

```text
Request Tracking

Distributed Tracing

Debugging
```

---

### Q3. What is the difference between Request ID and Correlation ID?

```text
Request ID

→ Single Service

Correlation ID

→ Multiple Services
```

---

### Q4. Where are Correlation IDs typically stored?

```java
MDC
```

---

### Q5. How are Correlation IDs displayed in logs?

```xml
%X{correlationId}
```

---

### Q6. Why is MDC used with Correlation IDs?

To automatically add correlation information to every log.

---

### Q7. Which HTTP header commonly carries Correlation IDs?

```http
X-Correlation-ID
```

---

### Q8. Should a new Correlation ID be generated in every service?

```text
No
```

The same ID should be propagated.

---

### Q9. Which is preferred for Correlation ID generation: Filter or Interceptor?

Generally:

```text
Filter
```

because it runs earlier.

---

### Q10. What is Correlation ID Propagation?

Passing the same Correlation ID across all services participating in a request.
