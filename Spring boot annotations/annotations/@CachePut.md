
## Definition

`@CachePut` is a Spring Cache annotation used to **update the cache with the method's return value**.

It tells Spring:

> Always execute the method and then store/update the result in the cache.

Package:

```java
import org.springframework.cache.annotation.CachePut;
```

---

# Prerequisite

Caching must be enabled.

```java
@Configuration

@EnableCaching
public class CacheConfig {
}
```

Without:

```java
@EnableCaching
```

`@CachePut` does nothing.

---

# Why Do We Need @CachePut?

Suppose data is already cached.

```java
@Cacheable("products")
public Product getProduct(Long id) {
    return repository.findById(id)
                     .orElse(null);
}
```

---

Cache:

```text
Key = 1
Value = Old Product
```

---

Now product gets updated.

```java
repository.save(updatedProduct);
```

---

Problem:

```text
Database Updated
Cache Still Contains Old Data
```

---

Result:

```text
Stale Data Returned
```

---

Solution:

```java
@CachePut
```

---

# What Does @CachePut Do?

Unlike:

```java
@Cacheable
```

which may skip method execution,

`@CachePut`:

```text
Always Executes Method
Always Updates Cache
```

---

# Basic Example

```java
@Service
public class ProductService {

    @CachePut(
      value = "products",
      key = "#product.id"
    )
    public Product updateProduct(
            Product product) {

        return repository.save(product);
    }
}
```

---

Execution:

```text
Method Called
      ↓
Database Updated
      ↓
Result Stored In Cache
      ↓
Return Result
```

---

# @CachePut vs @Cacheable

Interview Favorite.

---

## @Cacheable

```text
Check Cache First
```

---

Cache Hit:

```text
Method Not Executed
```

---

## @CachePut

```text
Always Execute Method
```

---

Then:

```text
Update Cache
```

---

Comparison:

| Feature | @Cacheable | @CachePut |
|----------|-----------|-----------|
| Check Cache First | ✅ | ❌ |
| Skip Method | ✅ | ❌ |
| Execute Method | Only Cache Miss | Always |
| Update Cache | Cache Miss | Always |
| Best For | Reads | Updates |

# Key Points To Remember

- `@CachePut` always executes the method.
- Updates cache using returned value.
- Requires `@EnableCaching`.
- Supports custom keys using SpEL.
- Supports `#result`.
- Does not skip method execution.
- Commonly used after update operations.
- Uses Spring AOP proxies internally.
- Helps keep cache synchronized with database.
- Frequently asked Spring Cache interview topic.

---

# Quick Comparison

| Annotation | Purpose |
|------------|----------|
| `@Cacheable` | Read From Cache |
| `@CachePut` | Update Cache |
| `@CacheEvict` | Remove Cache |
| `@Caching` | Combine Operations |
| `@EnableCaching` | Enable Caching |

# Interview Shortcut

```java
@CachePut(
 value = "products",
 key = "#product.id"
)
public Product updateProduct(
      Product product) {

    return repository.save(product);
}
```

Execution:

```text
Method Called
      ↓
Database Updated
      ↓
Cache Updated
      ↓
Return Result
```

Think of it as:

```java
Product product =
      repository.save(product);

cache.put(
    product.getId(),
    product
);

return product;
```

implemented automatically by Spring.

`@CachePut` is the Spring caching annotation used when data changes and the cache must be refreshed with the latest value while still executing the underlying method every time.
